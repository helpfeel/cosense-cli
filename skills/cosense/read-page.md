# Cosenseのページを読む手順書

CLI コマンドの仕様や実行形式は [SKILL.md](SKILL.md) を参照する。
各コマンド毎に戻り値のスキーマが異なるので、初めて使う前に必ず `cosense <command> --help` で確認する。

## 起点ページを決める

### URLが指定されている場合

「ページを読んでいく」へ進む。

### 何らかのテーマや問いがある場合

#### `searchVector` で少なくとも5回は検索する

短いクエリは短いタイトルに、長いクエリは長いタイトルにマッチしやすい

つまり、ユーザーの質問を元にした、ある程度の長さの自然文の入力は効果的である。
長く具体的なタイトルのページには、そのトピックに関する具体的な情報が書かれている傾向がある。

ユーザーの指示から考えた単語を複数入力するのも効果的である。
短く抽象的なタイトルのページには、重要な原理原則が書かれている事がある。

`score` と `title` を総合判断して起点候補を複数選ぶ。

`exists: false` の空ページも採用してよい。本文が無くても関連ページリストはあるから。

#### 良い起点が見つからない場合

`searchFullText` で本文に当該語を含むページを探す。

## ページを読んでいく

以下の手順1〜3は必ず順番に全て実行する。状況に応じて選ぶメニューではない。

### 手順1: ページを取得して読む

- `readPage` でそのページのデータを取得する
- `user`/`lastUpdateUser`/`users` で誰が書いたのかを意識する
- `created`/`updated` でいつ書かれたかを意識する
- `pageRank` / `linked` / `views` でページの注目度を把握する
- `lines[]` を本文として読む
- 本文が空または `persistent: false` なページであっても、他のページの文脈の中で説明されている抽象概念である場合もあるので、必ず手順2へ進む

### 手順2: 周辺ページの文脈を掴む

- `browseRelatedPages` で1-hop と 2-hop の関連ページタイトル一覧を眺め、このページが属する概念の文脈を掴む
- タイトルを眺めるだけでよい。気になるページがあれば次の手順で読む

### 手順3: 1-hop近傍を辿る

- `list1hopLinks` で1-hop近傍メタデータを一括取得する。テーマで絞りたい時は `search1hopLinks` を使う
- 各要素の `relation` で outgoing（そのページが参照しているページ）/ incoming（そのページを参照しているページ）/ bidirectional（双方向）の3種類が見分けられる
- `descriptions` を眺めて、ユーザーの問いに関連するもの・注目度の高いものを `readPage` で読む
- 全件読む必要はない。意図理解に必要な範囲で AI が判断する

## 完了条件と回答

- そのページの意図・背景が理解できた状態で終了する。「全リンク読破」を完了条件としない
- 複数ページを調査した場合、必要な情報源が揃ったら終了する。集めても回答精度が上がらないと判断したら、見つかった範囲で答える
- 情報源（URL）を示しながらユーザーの問いに答える
